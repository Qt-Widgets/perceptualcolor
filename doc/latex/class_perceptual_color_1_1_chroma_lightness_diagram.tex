\doxysubsection{Detailed Description}
A widget that displays a chroma-\/lightness diagram. 

This widget displays a chroma-\/lightness diagram in the L\+Ch color model for a given hue, in a carthesian coordinate system.

The widget shows the chroma-\/lightness diagram at the whole widget extend. At the y axis the diagram always shows the lightness values of {\ttfamily 0} at the bottom pixel and the lightness value of {\ttfamily 100} at the top pixel. For the {\ttfamily x} (chroma) axis is automatically used the same scale as for the {\ttfamily y} axis. So if the widget size is a square, both {\ttfamily x} range and {\ttfamily y} range are from {\ttfamily 0} to {\ttfamily 100}. If the widget width is larger than the widget height, the {\ttfamily x} range goes beyond {\ttfamily 100}.

The widget reacts on mouse events and on keyboard events (see \mbox{\hyperlink{class_perceptual_color_1_1_chroma_lightness_diagram_abd23775803f181201fefbdcc08378ab8}{key\+Press\+Event()}} for details).

Generates an image of a chroma-\/lightness diagram. 

This function generates images of chroma-\/lightness diagrams in the Lch color space. This function should be thread-\/save as long as you do not use the same Little\+C\+MS transform from different threads. (Also, out of the Qt library, it uses only Q\+Image, and not Q\+Pixmap, to make sure the result can be passed around between threads.)
